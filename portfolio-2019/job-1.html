Leroy Merlin Ecommerce

Omnichannel project through the ecommerce platform.

On this project:

User experience, user interface, web application

The challenges

The biggest challenge here was create a solution to the whole buy flow, including the product page, order tracking and checkout. All of those features should be an omnichannel solution.

Product Page

After to analyze the product page numbers with Google Analytics and Hotjar, we decided to talk with the business area to start a new product page project, but first we called a meeting and asked the business team to fill a spreadsheet with references about what is an ideal product page, and what kind of features we should have there.

We've done more 2 meetings to ask different teams to help us to create the new structure of a new page, with the new features and components, so we've got a room, post-its and a board to draw it.

After this fase, I designed a wireframe and then a high fidelity prototype to test the new features with some customers, to understand if we'd need to do some changes or not. Then I went to the store to test it out and check some questions we had about the new features, amount of the information on the same screen and asked the users to do few tasks like choose a product color, calculate the shipment price and add to cart.

The surprises

This project was my first with supply chain and the financial departments.

I've haver worked with a supply chain project before, so I had to learn a lot about logistics and their business rules. I had to learn about the suppliers reality and their pains too when a big company doesn't have a good solution to work with the suppliers and what kind of problem the final custmer can have, like a lack of products to buy, for example.

Today this project is a profit application, because the company has like different prices for any kind of service through the portal.
